# Docker管理界面简化报告

## 🎯 简化目标

根据用户反馈，Docker管理界面展示过于复杂，需要进行以下简化：
- 状态文字太长需要简短
- 端口展示过长需要优化
- 整体界面复杂度需要降低

## ✅ 完成的简化改进

### 1. 状态显示简化
**改进前**: "运行中"、"已停止"、"重启中"  
**改进后**: "运行"、"停止"、"重启"

```typescript
const getShortStatus = (state: string) => {
  const statusMap: Record<string, string> = {
    'running': '运行',
    'exited': '停止',
    'stopped': '停止',
    'paused': '暂停',
    'restarting': '重启',
    'dead': '异常',
    'created': '已创建'
  }
  return statusMap[state] || state
}
```

### 2. 端口显示简化
**改进前**: `0.0.0.0:8080 -> 80/tcp, 0.0.0.0:8443 -> 443/tcp`  
**改进后**: `8080:80, 8443:443`

```typescript
const formatPort = (port: any) => {
  if (!port.PublicPort) return '-'
  const pubPort = port.PublicPort
  const privPort = port.PrivatePort || '-'
  return `${pubPort}:${privPort}`
}
```

### 3. 表格列数减少
**改进前**: 8列 (选择框、状态、名称、镜像、CPU/内存、端口、创建时间、操作)  
**改进后**: 6列 (选择框、状态、名称、镜像、资源、端口、操作)

移除了"创建时间"列，减少视觉复杂度。

### 4. 列标题简化
**改进前**: "CPU / 内存"  
**改进后**: "资源"

### 5. 过滤选项简化
**改进前**: "运行中"、"已停止"  
**改进后**: "运行"、"停止"
